Site Header

There are 2 site headers: one for the site title and one for the nav bar. Adjust your window size as necessary.

This Template uses:

  • The Runner Object
  • The Mast Component
  • Constrain
  • The Float Grid
  • The Site Title
  • The Camo Link
  • The Toggle Button
  • The Drawer Object
  • The Mobile Menu
  • The Menu Bar

Highlighted Content Block

Highlighted Content Blocks, commonly referred to as callouts, are modular blocks of content movable anywhere on a page. They might contain a heading, some content, and one or more action buttons. Typically, the heading is given an id to be used as a fragment identifier (internal page anchors).

This Template uses:

  • The Runner Object
  • Constrain
  • The Action List
  • Buttons
  • The Banner Component (optional)

Hero Image

The Hero Image is the largest image on the home page, containing the current conference name and information, call-to-action buttons, and other patterns. The Hero Image Template also contains microdata HTML, which is unstyled and not considered part of any design component.

This Template uses:

  • The Runner Object
  • The Banner Component
  • The Conference Header
  • Constrain
  • The Page Title
  • The Action List
  • Buttons

Other Year Callout

Use the Other Year Callout to promote last cycle’s and next cycle’s conferences. This Template is very similar to the Hero Image, but with some differences:

  • The microdata is slightly different (but visually unnoticeable).
  • The conference dates and theme are ommitted, and replaced with a small blurb about why users should visit the page or site.
  • Being a highlighted content block, this Template does not use a Page Title as the Hero Image does.

This Template uses:

  • The Runner Object
  • The Banner Component
  • The Conference Header
  • Constrain
  • The Action List
  • Buttons

Person

The Person content type can be marked up many ways. We’ve already seen one way, which is the Speaker Template above. The other ways are listed below and do not use Components.

Each of these Template patterns should be wrapped with an element with the attributes [itemscope="" itemtype="http://schema.org/Person"].

This Template uses no other established patterns.

Place

A place can have a name, physical address, telephone, or url. Wrap with <address> only if the place represents contact information for the article or document author.

This Template pattern should be wrapped with an element with the attributes [itemscope="" itemtype="http://schema.org/Place"].

This Template uses no other established patterns.
