The Mast Component§

A Mast is a section or banner that runs across the full width of the page. There are four types of masts: head, main, sitemap, and foot. These four are instances of the generic Mast Component. In these examples, we use the div element, but it may be more appropriate to use header or footer on an actual site.

A generic mast. It uses the site’s Dark Gray background color.

The Mast Head contains the Site Title Component. It has a slight gradient: Dark Gray to Dark Gray t1, left to right.

The Mast Menu contains the Menu Bar Component. It has a slight gradient: Primary s1 to Primary, left to right.

The Mast Sitemap contains the Sitemap Component. It uses the Primary background color, and its links are underlined.

The Mast Foot contains the copyright line and ASCE contact info. Its text-align is centered and it also has a slight gradient: Dark Gray to Dark Gray t1, top to bottom. Its links are also underlined.

The Banner Component§

The Banner Component can be used to give a Runner a background image.

Subclasses of Banner:

  • .c-Banner--hero — banner image on home page
  • .c-Banner--orgs — banner for additional organizations
  • .c-Banner--location — banner image for location page title
  • .c-Banner--prev — previous year highlight block (blurred)
  • .c-Banner--next — next year highlight block (blurred)
  • .c-Banner--loc — location highlight block (same as location title, but blurred)

The Program Heading§

The Program Heading is a date that serves as the heading of a The Time Block. It is of the format Ddddddd, Mmm DD. The first one is normal, the second one is “selected”—after an input that is checked.

Program Headings inside a The Banner Component have different colors.

The Supporter Block§

A Supporter Block holds one group of supporters.

On the Sponsor page, each Supporter Block corresponds to one level, e.g., Platinum, Gold, or Corporate. On all other pages, this block may be used to group additional organizations, such as Cooperating Organizations. This block is a section containing a heading (the name of the level) followed by an unordered list of supporter logos.

To separate multiple supporter blocks, add class .c-SupporterBlock--separate for a thin bottom border.

Supporter Blocks, regardless of display, hold logos for sponsoring or supporting organizations. Typically, they are links with background images, containing hidden text for accessibility.

There are three subclasses of .c-SupporterBlock__Logo:

  • .c-SupporterBlock__Logo--sml
  • .c-SupporterBlock__Logo--med
  • .c-SupporterBlock__Logo--lrg

The Social List§

A generic Social List Component is shown below. The generic form of this Component shouldn’t be used on its own; there are three flavors of Social List shown below.

The first type of Social List is meant for the site mast footer.

The second type, similar to the foooter, belongs in the Speaker Component.

The third is meant for a highlighted content block. The logos are larger and each encompassed in a circle. Logos in a Banner are slightly lighter.

The available subclasses of .c-SocialList__Link are listed below. Each subclass corresponsds to its designated social network.

  • .c-SocialList__Link--twitter (Twitter)
  • .c-SocialList__Link--facebook (Facebook)
  • .c-SocialList__Link--google (Google+)
  • .c-SocialList__Link--linkedin (LinkedIn)
  • .c-SocialList__Link--youtube (YouTube)

There are also links available for traditional contact information.

  • .c-SocialList__Link--email (email address)
  • .c-SocialList__Link--phone (telephone number)
  • .c-SocialList__Link--explore (homepage / website)
  • .c-SocialList__Link--phone_iphone (mobile phone)

Buttons§

Buttons are displayed inline-block, so in order to preserve vertical rhythm, be sure to contain them in typographical block-level elements such as paragraphs or lists.

Button Components may be in the form of a elements (links or anchors) or button elements. Keep in mind these button usage guidelines: buttons should indicate action and be worded actionably. Use an a instead of a button if there is a meaningful target href or url. Of course, a elements as well as button elements may be styled to look like Button Components.

Every Button Component must have styles available for each of the following states:

default
the button does something or the link goes somewhere, but the user is not interacting with the component
hover/focus
the user’s cursor is hovering over the component, or the user agent is focusing on the component
active
the user is in the process of clicking/activating the component
disabled
the component is inaccessable and/or does nothing, and the user may not interact with it. this state is implemented by
  • a button with the disabled attribute, or
  • an a with the href attribute absent (this is not the same as an empty attribute [href=""]).

Each of these states is demonstrated in the sections below, however only the code for the default state is shown.

Important: colors will vary by site.

The Camo Link§

A Camo Link is a link that blends with its surrounding text. It unsets the default styles assigned to unclassed a elements (also known as anchors). You should use it when you need a hyperlink but don’t want it visually exposed to the user. In fact, this class can be used on any element, not just anchors.

Note to developers: the .c-LinkCamo class uses !important on all its styles, so use only when necessary. And if possible, use on the innermost element. E.g.:

<span><a class="c-LinkCamo">correct</a></span>
<!-- a class="c-LinkCamo"><span>incorrect</span></a -->

The Caps Link§

A Caps Link is a link styled slightly bolder and with all caps. It is a similar style to links shown in the menus. You can use this style as an alternative to a button style, for example, in a dialogue box. This class can be used on any element, not just anchors.
