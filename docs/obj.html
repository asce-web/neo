The Runner Object§

The Runner Object is a large, full-width section with padding on all four sides. It can be used for headers, footers, heros, highlight blocks, etc., really, anything that looks like a “row” of a page. Change the window size and watch what happens to the padding.

Subclasses of the Runner Object simply vary the padding.

Use .o-Runner--pageHeader for headers of pages, whether they be generic pages or special pages with hero banner headers.

Use .o-Runner--highlight for highlighted content callouts.

Constrain§

This object wraps content, limiting its maximum width. Use it to wrap content directly inside a Runner (though it may not necessarily be a child of Runner).

The Float Grid§

The Float Grid is a list of items arranged in a grid-like structure. It is similar to the Grid Object, except this object uses floating items instead of the Flexbox model. It works in older browsers such as Internet Explorer 9.

The Action List§

The Action List contains a variable number of items (typically 3 or fewer). Most commonly these items are actionable elements, such as links or buttons. However, this Object does not discriminate its contents—it can be used to hold static items as well. The list items are stacked vertically by default and become horizontal on Medium screens (720px) and larger. When displayed horizontally, the items have equal width (unlike Flex Object items, whose widths are automatically sized).

The Stacked List§

The Stacked List Object is usually an unordered list (though it can be ordered) of blocky items. These items may be sponsor logos, passes, speakers, and more. This Object is responsive, adjusting its items’ widths at different breakpoints.

The Tabs Object§

The Tabs Object contains a series of corresponding tabs and panels. It is similar to a carousel, but only one panel is shown at a time and there is no timed automatic progression. Each Tabs Object must have a unique id for the carousel effect to work properly.

Enable Javascript to view animation effects.
